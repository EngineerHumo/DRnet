Overall objective for GSE160306 Python bioinformatics analysis (updated)

一、总目标

只基于 GSE160306，完成一套纯 Python 的可复现生信分析流程，用于 diabetic retinopathy（DR）炎症相关候选基因研究，并输出可直接用于论文撰写的结果表、统计结果、图像和总总结文件。

本项目当前已经具备以下 gene set 文件：
1. data_raw/hallmark_human_gene_symbols.gmt
   - 官方 Hallmark gene sets 总文件（50 个 Hallmark gene sets）
   - 后续 GSEA、通路富集和基因集提取均以此文件为主来源
2. data_raw/hallmark_inflammatory_response.gmt
   - 单独提取出的 HALLMARK_INFLAMMATORY_RESPONSE gene set
   - 可直接用于 inflammation ssGSEA
   - 若脚本中已支持从 hallmark_human_gene_symbols.gmt 动态提取该 gene set，则此文件可作为冗余备用文件
3. data_raw/immune_28_signatures_tisidb.gmt
   - 当前内容为 28 immune cell signatures，可用于 ssGSEA
   - 但文件内部描述更接近基于 Charoentong 2017 / TISIDB 整理版本，而非严格意义上的 TISIDB 原始官方命名版本
   - 必须先完成名称清洗和标准化后，再作为正式分析输入文件

二、必须先改掉的旧逻辑

1. 不允许沿用当前 metadata.py 的早期二分类逻辑。
   该脚本会过早把样本压成 Control/DR，并丢失 diabetic 和 NPDR 两个中间层级，破坏四组严重度分析。

2. 不允许直接用 normalized CPM 做差异分析。
   差异分析必须基于 raw counts。
   normalized CPM 仅用于 log2(CPM+1) 后的 PCA、heatmap、ssGSEA、LASSO 和表达可视化。

3. 不允许再用“Pearson 定义 DE-INFGs”作为主口径。
   正式候选基因定义统一为：
   primary DEGs ∩ HALLMARK_INFLAMMATORY_RESPONSE gene set

4. 不允许硬编码旧稿中的结果数字。
   包括但不限于 98、45、7、38/20 等，全部必须由新流程重新计算。

5. IPA 不作为 Python 自动分析步骤。
   Python 主流程只导出 ipa_input_selected_genes.csv，供后续人工上传 IPA。

6. immune 28 signatures 不允许直接把原始文件名当作来源证明。
   必须先检查并清洗 immune_28_signatures_tisidb.gmt 内部 gene set 名称，包括：
   - 修正拼写错误（如 memeory -> memory）
   - 去除多余空格
   - 将 gene set 名统一转为适合代码和图表使用的标准命名
   - 输出正式清洗后的文件供后续 ssGSEA 使用

四、分析设计总原则

1. 先从 GSE160306 series matrix 解析完整 metadata，保留四组原始标签：
   healthy control
   diabetic
   NPDR
   NPDR/PDR + DME

2. 只保留 macula 样本进入正式分析。

3. 主比较：
   healthy control vs NPDR/PDR + DME

4. 同时保留四组严重度/进展支持分析：
   healthy control = 0
   diabetic = 1
   NPDR = 2
   NPDR/PDR + DME = 3

5. 炎症候选基因主定义：
   primary DEGs ∩ HALLMARK_INFLAMMATORY_RESPONSE

6. progressive inflammatory genes 定义：
   炎症候选基因中，同时满足四组严重度趋势显著的基因

7. LASSO 仅在主比较组中做，使用 repeated stratified cross-validation。

8. 组合 ROC 必须使用 out-of-fold predicted probabilities，不允许用训练集拟合概率直接报告。

9. immune infiltration 只作为 supportive evidence，不允许表述为真实浸润绝对定量。

五、gene set 文件的正式处理要求

1. hallmark_human_gene_symbols.gmt
   - 作为 Hallmark 总文件保留原样
   - 后续 GSEA 统一使用此文件

2. hallmark_inflammatory_response.gmt
   - 检查是否只有一行且对应 HALLMARK_INFLAMMATORY_RESPONSE
   - 如果格式正确，可以直接用于 inflammation ssGSEA
   - 同时允许脚本从 hallmark_human_gene_symbols.gmt 动态提取同一 gene set 做一致性检查

3. immune_28_signatures_tisidb.gmt
   - 先检查是否确实包含 28 个 gene sets
   - 检查每个 gene set 是否为合法 GMT 格式
   - 统一清洗 gene set 名称
   - 输出正式文件：
     data_processed/immune_28_signatures_charoentong2017_human_cleaned.gmt
   - 另输出 gene set 来源和清洗日志：
     results/logs/gene_set_source_and_cleaning_log.md

六、各脚本职责

00_config.py
- 路径、参数、阈值、随机种子、分组名称、绘图设置统一管理

00a_validate_and_clean_gene_sets.py
- 检查三个 gmt 文件是否合法
- 从 hallmark_human_gene_symbols.gmt 中确认 HALLMARK_INFLAMMATORY_RESPONSE 是否存在
- 检查 hallmark_inflammatory_response.gmt 是否与主文件中对应 gene set 一致
- 检查 immune_28_signatures_tisidb.gmt 是否为 28 个 gene sets
- 清洗 immune gene set 名称
- 输出 cleaned immune gmt 和清洗日志

01_parse_manifest.py
- 解析 series matrix
- 保留所有 metadata 字段
- 输出 all samples / macula 4 groups / primary binary 三套 manifest

02_prepare_expression.py
- 读取 counts 和 normalized CPM
- 对齐样本
- 基因 ID 处理
- 生成 raw counts 和 log2CPM 分析矩阵

03_qc_and_pca.py
- library size
- expression distribution
- sample correlation
- clustering
- PCA

04_differential_expression.py
- primary differential expression
- diabetic vs healthy
- NPDR vs healthy
- severity trend analysis

05_inflammation_scoring.py
- 用 hallmark_inflammatory_response.gmt 做 inflammation ssGSEA
- 主比较与四组趋势统计

06_candidate_selection.py
- primary DEGs 与 inflammatory response 取交集
- 结合 severity trend 生成 progressive inflammatory genes

07_lasso_signature.py
- 基于候选基因进行 LASSO 筛选
- 输出单基因 ROC 和组合 signature ROC
- 输出 OOF probabilities、signature score、系数表

08_enrichment_analysis.py
- 基于 selected genes 做 Spearman-ranked GSEA
- 使用 hallmark_human_gene_symbols.gmt 及其他可用 gene sets
- 导出 ipa_input_selected_genes.csv

09_immune_infiltration.py
- 使用 cleaned immune 28 signatures gmt 做 ssGSEA
- 输出 immune score、组间差异、趋势分析、gene-immune correlation

10_make_figures.py
- 统一生成主图和补图

11_export_master_summary.py
- 汇总所有结果
- 输出 master_analysis_summary.csv 和 master_analysis_summary.md

run_all.py
- 串联运行整个流程

七、最终必须输出的核心结果

1. 样本清单与分组文件
2. 预处理表达矩阵
3. DEG 结果表
4. inflammation score 结果表
5. inflammatory_core_genes 与 progressive_inflammatory_genes 清单
6. LASSO 入选基因和 signature score
7. GSEA 结果表
8. immune ssGSEA 结果表
9. 主图、补图
10. IPA 输入文件
11. master summary
